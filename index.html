Matiz (Hue)

A Matiz é a cor em seu estado puro, como o "vermelho", "verde" ou "azul". Ela é representada por um ângulo de 0 a 360 graus no círculo cromático. Por exemplo: 0° é vermelho, 120° é verde e 240° é azul. Alterar a matiz é como girar o seletor de cores para escolher uma tonalidade completamente nova, sem afetar seu brilho ou intensidade.

Luminosidade (Lightness)

A Luminosidade define o quão clara ou escura uma cor é. É uma porcentagem que vai de 0% (preto total) a 100% (branco total). O valor de 50% representa a cor em sua claridade "normal". Mudar a luminosidade é útil para criar versões mais escuras (sombras) ou mais claras (tons) de uma mesma cor base.

Saturação (Saturation)

A Saturação descreve a intensidade ou a "pureza" de uma cor. Também é representada por uma porcentagem, indo de 0% (totalmente cinza) a 100% (a cor mais vibrante e pura possível). Diminuir a saturação torna a cor mais opaca e acinzentada, enquanto aumentá-la a deixa mais viva e rica.

HSL vs. RGB/Hex

Enquanto o HSL é projetado para ser intuitivo para a percepção humana, os modelos RGB e Hexadecimal funcionam de maneira diferente:

  • RGB (Red, Green, Blue): Descreve uma cor pela mistura de luz vermelha, verde e azul (escala de 0 a 255 para cada). É como as telas de computadores e TVs fisicamente produzem as cores. No entanto, é pouco intuitivo para fazer ajustes finos. Por exemplo, para deixar um azul mais claro, você precisa aumentar os valores de vermelho e verde, o que não é um processo óbvio.

  • Hex (Hexadecimal): É apenas uma forma mais compacta de escrever o código RGB, usando a notação hexadecimal (ex: #0000FF para azul). É muito comum na web, mas sofre da mesma falta de intuitividade do RGB para manipulação humana.

A grande vantagem do HSL é permitir que você ajuste a claridade ou a intensidade de uma cor (Luminosidade e Saturação) sem precisar recalcular os três valores de cor, como seria necessário no RGB.
